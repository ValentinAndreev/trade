# Product Overview

## Цель

`Trade Data Analysis` это рабочее место для анализа рыночных данных внутри браузера. Приложение позволяет следить за набором криптоинструментов, просматривать связанные рынки, строить графики, рассчитывать индикаторы, собирать табличные выборки и проверять торговые гипотезы на истории.

## Ключевые возможности

### 1. Рыночный обзор

На главной странице отображаются:

- криптовалютные тикеры из Bitfinex;
- мировые индексы;
- валютные пары;
- commodities.

Криптовалютный блок ориентирован на список отслеживаемых торговых символов. Рыночные плитки по индексам, forex и commodities служат как контекстный дашборд и получают котировки отдельно.

### 2. Графики

Раздел `Graph` поддерживает:

- несколько табов;
- несколько панелей внутри таба;
- несколько оверлеев на панели;
- переключение таймфреймов;
- price/volume/indicator режимы;
- макро-оверлеи (DXY, VIX, Fear & Greed, Fed Rate, M2, CPI) с отдельной шкалой;
- фильтр источников индикаторов: Client / Server / Data Sources;
- пользовательские метки и графические примитивы;
- realtime-обновление по WebSocket.

### 3. Табличный анализ

Пользователь может собрать data tab с историческими свечами и вычисляемыми столбцами:

- OHLCV;
- индикаторы (client / server);
- процентные и абсолютные изменения;
- формулы;
- колонки по другим инструментам;
- макро-данные (DXY, VIX, Fear & Greed, Fed Rate, M2, CPI) с LOCF-интерполяцией.

### 4. Conditions и Trading Systems

На базе таблицы можно:

- заводить условия для фильтрации и подсветки строк;
- переносить сигналы на график;
- описывать правила входа и выхода;
- получать системную статистику и equity curve.

### 5. Presets

Приложение умеет сохранять рабочее пространство пользователя:

- набор табов (chart, data, system_stats, research/Test/Optimization, system_editor, assistant);
- активный таб;
- список dashboard-symbols;
- список market-symbols.

### 6. LLM-ассистент

Встроенный AI-ассистент для разработки торговых систем.

Возможности:

- диалог с контекстом текущей системы из YAML-редактора;
- генерация и патч торговых систем в режимах `system_design` / `system_patch`;
- сохранение истории диалогов в БД;
- поддержка провайдеров: OpenAI, Anthropic, Gemini, OpenRouter, Mistral, xAI, Ollama, llama.cpp.

API-ключи провайдеров вводятся через UI и хранятся в БД в зашифрованном виде.

## Рабочее пространство

Приложение работает в едином рабочем пространстве.

Когда табов нет — показывается дашборд быстрого мониторинга: крипто-тикеры, индексы, forex, commodities. Клик по крипто-тикеру открывает chart tab.

Когда есть хотя бы один таб — дашборд скрывается, пространство занимают табы. Слева графики и таблицы, справа боковая панель настроек активного элемента. Таб-бар с кнопками добавления всегда виден.

## Основные сущности

- `Candle`: историческая свеча по инструменту и таймфрейму.
- `MacroSeries`: макроэкономическая временная серия (VIX, DXY, Fear & Greed, Fed Rate, M2, CPI).
- `Preset`: сохраненное пользовательское состояние рабочего пространства.
- `Tab`: вкладка рабочего интерфейса.
- `Panel`: подпанель внутри графического таба.
- `Overlay`: слой на графике, например price, volume, indicator или macro.
- `Condition`: правило для поиска событий в табличных данных.
- `TradingSystem`: набор правил входа и выхода для расчета метрик.
- `LlmSetting`: настройки LLM-провайдера пользователя с зашифрованным API-ключом.
- `AiChat`: диалог с LLM-ассистентом, хранящий историю сообщений.

Подробно: [03 Domain Model](03-domain-model.md)

## Что считать "истиной" в данных

- История свечей и агрегаты находятся в PostgreSQL/TimescaleDB.
- Макро-данные (FRED, Yahoo Finance, AlternativeMe) синхронизируются в TimescaleDB через `MacroSyncJob` и являются server-side source of truth.
- Текущие live tickers по crypto сначала берутся из Bitfinex API, при недоступности возможен fallback к БД.
- Рыночные плитки по индексам, forex и commodities берутся из Yahoo Finance и не записываются в БД как полноценная историческая серия.
- История диалогов LLM-ассистента и настройки провайдеров хранятся в PostgreSQL.
